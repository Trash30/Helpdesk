2

Installation automatique (recommandée)

Le projet inclut deux scripts qui automatisent l'installation des prérequis et la configuration complète en quelques minutes.

⚠️
Droits administrateur requis Les deux scripts ci-dessous doivent être exécutés en tant qu'administrateur (clic droit → "Exécuter en tant qu'administrateur").

2.1   Étape 1 — Installer Node.js et PostgreSQL

Ce script télécharge et installe silencieusement Node.js 20 LTS et PostgreSQL 15.

  • Ouvrir un terminal en tant qu'administrateur Appuyer sur Win + X → "Terminal Windows (admin)" ou rechercher "cmd" → clic droit → "Exécuter en tant qu'administrateur".
  • Se placer dans le dossier du projet Remplacez le chemin par l'emplacement réel du projet sur votre machine.
  • Lancer le script d'installation des prérequis Double-clic sur local-prereqs.bat ou l'exécuter depuis le terminal. Durée estimée : 3 à 8 minutes selon la connexion. 
# Depuis un terminal administrateur dans le dossier du projet :
.\local-prereqs.bat

# Le script installe automatiquement :
#   ✓ Node.js 20.14.0 LTS (MSI silencieux)
#   ✓ PostgreSQL 15 (installateur EDB silencieux)
#   ✓ Mise à jour du PATH système

2.2   Étape 2 — Configurer et initialiser le projet

Ce script configure la base de données, crée le fichier .env, installe les dépendances npm et charge les données de démarrage.

  • Fermer et rouvrir le terminal Nécessaire pour que les nouvelles variables PATH (node, psql) soient prises en compte.
  • Double-cliquer sur local-setup.bat Ce script n'a pas besoin de droits admin. Il peut être exécuté en simple double-clic.
  • Répondre aux questions de configuration Hôte PostgreSQL, port, utilisateur, mot de passe, nom de la base de données, nom de l'entreprise. Appuyez sur Entrée pour accepter les valeurs par défaut. 
# Le script vous demande :
  Hote PostgreSQL [localhost]    → Entrée (défaut)
  Port PostgreSQL [5432]        → Entrée (défaut)
  Utilisateur     [postgres]     → Entrée (défaut)
  Mot de passe                  → Votre mot de passe PostgreSQL
  Nom base        [helpdesk_dev] → Entrée (défaut)
  Nom société     [Mon Helpdesk] → Votre nom (optionnel)

# Puis automatiquement :
#   ✓ Création de la base helpdesk_dev
#   ✓ Génération d'un JWT_SECRET aléatoire
#   ✓ Écriture de server/.env
#   ✓ npm install (server + client + racine)
#   ✓ prisma migrate deploy
#   ✓ prisma db seed (comptes de test)
✅
Setup terminé ! Quand le script affiche "Setup terminé ! Lancez local-start.bat pour démarrer", passez directement à la section 4.

Installation manuelle (étape par étape)

Suivez cette section si vous préférez contrôler chaque étape ou si les scripts automatiques ont échoué.

3.1   Installer Node.js 20 LTS

  • Télécharger l'installateur Rendez-vous sur nodejs.org → "Downloads" → sélectionnez la version 20.x LTS pour Windows (x64).
  • Exécuter l'installateur .msi Acceptez les conditions. Laissez toutes les options par défaut cochées — notamment "Add to PATH" et "npm package manager".
  • Vérifier l'installation Ouvrez un nouveau terminal et exécutez les commandes suivantes : 
node -v     # Attendu : v20.x.x
npm  -v     # Attendu : 10.x.x ou plus

3.2   Installer PostgreSQL 15

  • Télécharger l'installateur EDB Rendez-vous sur postgresql.org/download/windows et téléchargez PostgreSQL 15 (EDB installer, Windows x86-64).
  • Lancer l'installation Exécutez l'installateur. Conservez le répertoire par défaut. Choisissez un mot de passe pour le super-utilisateur postgres — notez-le précieusement.
  • Ajouter PostgreSQL au PATH Ouvrez "Variables d'environnement système" → Path → Nouveau → ajoutez : C:\Program Files\PostgreSQL\15\bin
  • Vérifier l'installation Dans un nouveau terminal : 
psql --version   # Attendu : psql (PostgreSQL) 15.x

3.3   Créer la base de données

# Se connecter à PostgreSQL en tant que superuser :
psql -U postgres

# Dans le prompt psql, créer la base :
postgres=# CREATE DATABASE helpdesk_dev;
postgres=# \q    -- quitter psql

3.4   Créer le fichier server/.env

Créez un fichier nommé .env dans le dossier server/ avec le contenu suivant :

📄 server/.env 
# ── Base de données ───────────────────────────
DATABASE_URL=postgresql://postgres:VOTRE_MOT_DE_PASSE@localhost:5432/helpdesk_dev

# ── Authentification ──────────────────────────
# Générer un secret fort : openssl rand -base64 32
JWT_SECRET=remplacez-par-un-secret-aleatoire-fort

# ── Serveur ───────────────────────────────────
PORT=3001
NODE_ENV=development
APP_URL=http://localhost:5173

# ── Uploads ───────────────────────────────────
UPLOADS_PATH=./uploads

# ── SMTP (optionnel — voir section 6) ─────────
SMTP_HOST=
SMTP_PORT=587
SMTP_USER=
SMTP_PASS=
SMTP_FROM=noreply@helpdesk.local
COMPANY_NAME=Mon Helpdesk
🔐
Remplacez VOTRE_MOT_DE_PASSE et JWT_SECRET Ne laissez jamais les valeurs d'exemple en place. Le JWT_SECRET doit être une chaîne aléatoire d'au moins 32 caractères.

3.5   Installer les dépendances npm

# Depuis la racine du projet :
npm install                       # Racine (concurrently)
npm install --prefix server      # Backend
npm install --prefix client      # Frontend

3.6   Migrations et données initiales

# Depuis le dossier server/ :
cd server
npx prisma generate              # Générer le client Prisma
npx prisma migrate deploy        # Appliquer les migrations
npx prisma db seed               # Charger les données de test
cd ..
ℹ️
En cas d'erreur sur migrate deploy Si migrate deploy échoue (base vide, première fois), essayez npx prisma migrate dev --name init à la place.

Démarrer l'application

Démarrage automatique (recommandé)

Double-cliquez sur local-start.bat à la racine du projet. Ce script lance le backend et le frontend en parallèle dans la même fenêtre.

# Ou depuis un terminal :
.\local-start.bat

# Ce qui démarre :
#   Backend  (Express + Prisma) → http://localhost:3001
#   Frontend (Vite + React)    → http://localhost:5173

Démarrage manuel

Si vous préférez deux terminaux séparés :

💻 Terminal 1 — Backend 
cd server
npm run dev
# Attendu : "Server running on port 3001"
💻 Terminal 2 — Frontend 
cd client
npm run dev
# Attendu : "Local: http://localhost:5173/"

🚀 Application démarrée

🌐 http://localhost:5173
Backend API : http://localhost:3001
Fermer : Ctrl + C dans le terminal pour arrêter les serveurs

Configuration optionnelle — Emails SMTP

La configuration SMTP permet l'envoi automatique d'emails : réinitialisation de mot de passe et envoi d'enquêtes de satisfaction. Sans SMTP, ces fonctionnalités sont silencieusement désactivées.

Modifiez les variables suivantes dans server/.env :

📄 server/.env — Variables SMTP 
SMTP_HOST=smtp.votredomaine.com
SMTP_PORT=587                   # 587 (STARTTLS) ou 465 (SSL)
SMTP_USER=noreply@votredomaine.com
SMTP_PASS=votre-mot-de-passe-smtp
SMTP_FROM="HelpDesk <noreply@votredomaine.com>"
💡
Tester avec Mailtrap (développement) Pour tester les emails sans envoyer de vrais messages, créez un compte gratuit sur mailtrap.io et copiez les identifiants SMTP de votre boîte de réception fictive.

Après modification du .env, redémarrez le backend pour appliquer les changements.

Résolution des problèmes courants

❌ node ou npm introuvable après installation

Le PATH n'a pas été mis à jour. Fermez et rouvrez votre terminal, ou redémarrez Windows.

# Vérifier manuellement :
where node   # Doit afficher C:\Program Files\nodejs\node.exe

❌ psql introuvable

Le dossier bin de PostgreSQL n'est pas dans le PATH. Ajoutez manuellement :

# Ajouter temporairement au PATH de session :
$env:Path += ";C:\Program Files\PostgreSQL\15\bin"
# Ou ajoutez définitivement via les Variables d'environnement système

❌ Erreur de connexion PostgreSQL (ECONNREFUSED)

Le service PostgreSQL n'est pas démarré. Démarrez-le :

# Via les Services Windows (services.msc) :
# Chercher "postgresql-x64-15" → clic droit → Démarrer

# Ou via PowerShell (admin) :
Start-Service postgresql-x64-15

❌ Erreur d'authentification PostgreSQL (password authentication failed)

Le mot de passe dans DATABASE_URL ne correspond pas à celui du compte postgres. Vérifiez ou réinitialisez le mot de passe :

# Réinitialiser le mot de passe postgres :
psql -U postgres
postgres=# ALTER USER postgres PASSWORD 'nouveau-mot-de-passe';
# Puis mettre à jour DATABASE_URL dans server/.env

❌ Port 3001 ou 5173 déjà utilisé

# Trouver quel processus utilise le port :
netstat -ano | findstr :3001
taskkill /PID <PID> /F

# Ou modifier le port dans server/.env (PORT=3002)
# et dans client/.env.local (VITE_API_URL=http://localhost:3002)

❌ Erreur Prisma P1001 — Impossible de se connecter

# Vérifier que la DATABASE_URL est correcte dans server/.env
# Format : postgresql://USER:PASSWORD@HOST:PORT/DBNAME
cd server
npx prisma db pull    # Test de connexion

❌ Page blanche ou erreur CORS dans le navigateur

Le backend n'est pas démarré ou n'est pas accessible. Vérifiez que les deux terminaux tournent sans erreur et que http://localhost:3001/api/auth/me répond (devrait renvoyer 401).

Installation réussie Votre instance HelpDesk est opérationnelle sur http://localhost:5173. Pour toute question ou anomalie, consultez les logs dans le terminal de démarrage.
