Introduction

Objectif et perimetre de ce guide

Ce guide couvre le deploiement complet de l'application HelpDesk sur un mini PC ou serveur sous Ubuntu 22.04 LTS.

⚙️
Stack technique Node.js 20 + PostgreSQL 16 + PM2 (process manager) + Nginx (reverse proxy)
⏰
Duree estimee 30 a 45 minutes pour un deploiement complet.
1

Prerequis systeme

Mise a jour du systeme et installation des outils

Connectez-vous en SSH a votre serveur et executez les commandes suivantes :

# Mise a jour du systeme
sudo apt update && sudo apt upgrade -y

# Node.js 20 via nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install 20 && nvm use 20 && nvm alias default 20

# PostgreSQL, Nginx, Git
sudo apt install -y postgresql postgresql-contrib nginx git

# PM2 (process manager)
npm install -g pm2
🛈
Verification Apres l'installation, verifiez avec : node -v, npm -v, psql --version, nginx -v
2

Base de donnees PostgreSQL

Creation de l'utilisateur et de la base
sudo -u postgres psql <<EOF
CREATE USER helpdesk_user WITH PASSWORD 'ChangeMe123!';
CREATE DATABASE helpdesk_db OWNER helpdesk_user;
GRANT ALL PRIVILEGES ON DATABASE helpdesk_db TO helpdesk_user;
EOF
⚠️
Securite Remplacez ChangeMe123! par un mot de passe fort et unique. Ne reutilisez jamais ce mot de passe ailleurs.
3

Cloner le projet

Recuperation du code source
cd /opt
sudo git clone https://github.com/Trash30/Helpdesk.git helpdesk
sudo chown -R $USER:$USER /opt/helpdesk
cd /opt/helpdesk
4

Configuration — fichier .env

Variables d'environnement du serveur

Creez le fichier /opt/helpdesk/server/.env avec le contenu suivant :

DATABASE_URL=postgresql://helpdesk_user:ChangeMe123!@localhost:5432/helpdesk_db
JWT_SECRET=SECRET-ALEATOIRE-32-CARACTERES-MINIMUM
PORT=3001
NODE_ENV=production
UPLOADS_PATH=/opt/helpdesk/uploads
APP_URL=http://VOTRE-IP-OU-DOMAINE

# Email (optionnel)
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=
SMTP_PASS=
SMTP_FROM="Helpdesk <noreply@example.com>"
⚠️
JWT_SECRET Generez un secret aleatoire avec : openssl rand -hex 32
💡
APP_URL Remplacez par votre IP publique ou votre nom de domaine (ex: https://helpdesk.mondomaine.com).
5

Installation des dependances et build

npm install, migration, seed, build
cd /opt/helpdesk

# Installation des dependances
npm install --prefix server
npm install --prefix client

# Migration base de donnees + seed
cd server && npx prisma migrate deploy && npx prisma db seed && cd ..

# Build frontend + backend
npm run build --prefix client
npm run build --prefix server
✅
Verification Le build est reussi si aucun message d'erreur n'apparait. Les fichiers compiles du frontend se trouvent dans client/dist/.
6

Demarrage avec PM2

Lancement et persistance du serveur
cd /opt/helpdesk
pm2 start ecosystem.config.js
pm2 save
pm2 startup   # copier-coller la commande affichee
🛈
Verification Executez pm2 status pour voir l'etat du processus, et pm2 logs helpdesk-server pour consulter les logs en temps reel.
7

Configuration Nginx

Reverse proxy et serveur de fichiers statiques
# Copier la config Nginx fournie
sudo cp /opt/helpdesk/nginx.conf /etc/nginx/sites-available/helpdesk

# Editer server_name si vous avez un domaine
sudo nano /etc/nginx/sites-available/helpdesk

# Activer le site
sudo ln -s /etc/nginx/sites-available/helpdesk /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# Tester et recharger
sudo nginx -t && sudo systemctl reload nginx
sudo systemctl enable nginx
✅
Test Ouvrez http://IP-DU-SERVEUR dans un navigateur. Vous devez voir la page de login HelpDesk.
8

Exposition sur internet

Rendre l'application accessible depuis l'exterieur

Option A — Redirection de port (box internet)

Accedez a l'interface d'administration de votre box (ex: 192.168.1.1) et configurez une regle NAT/PAT :

Parametre Valeur
Protocole TCP
Port externe 80
IP locale IP du mini PC (ex: 192.168.1.50)
Port interne 80

Acces : http://VOTRE-IP-PUBLIQUE

Option B — Cloudflare Tunnel (recommande)

💡
Avantages SSL gratuit, aucun port a ouvrir, protection DDoS incluse. 
# Installation de cloudflared
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared-linux-amd64.deb

# Configuration
cloudflared tunnel login
cloudflared tunnel create helpdesk
cloudflared tunnel route dns helpdesk votre-sous-domaine.votre-domaine.com

# Installation en service
sudo cloudflared service install
9

SSL avec Let's Encrypt

Certificat HTTPS gratuit (si vous avez un domaine)
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d votre-domaine.com
sudo systemctl enable certbot.timer
🔒
Renouvellement automatique Le timer certbot.timer renouvelle automatiquement le certificat avant expiration.
10

Mises a jour

Procedure de mise a jour de l'application

Un script de deploiement est fourni a la racine du projet :

cd /opt/helpdesk && bash deploy.sh
🛈
Contenu du script Le script deploy.sh effectue automatiquement : git pull, installation des dependances, migration de la base, rebuild et redemarrage PM2.
11

Compte administrateur par defaut

Premiere connexion

Apres le seed initial, les identifiants par defaut sont definis dans le fichier server/prisma/seed.ts.

⚠️
Action obligatoire Changez le mot de passe administrateur immediatement apres la premiere connexion via la page Profil.
🎉
Deploiement termine ! Si toutes les etapes sont completes, votre instance HelpDesk est en production et accessible. Bonne utilisation !
