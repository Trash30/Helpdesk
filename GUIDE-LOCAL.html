Prérequis

Vérifications avant de commencer
ℹ️
Node.js déjà installé sur cette machine Node.js v24.14.0 et npm v11.9.0 ont été installés lors du dernier audit. La commande node --version doit répondre depuis un terminal PowerShell.

1.1 Vérifier Node.js et npm

Ouvrez un terminal PowerShell (Win + X → Terminal Windows) et exécutez :

node --version    # Attendu : v20.x.x ou v24.x.x
npm  --version    # Attendu : 10.x.x ou 11.x.x
⚠️
Si node est introuvable dans PowerShell Node.js est installé mais pas dans le PATH système. Fermez et rouvrez le terminal, ou redémarrez Windows. Si le problème persiste, lancez local-prereqs.bat en administrateur depuis le dossier du projet.

1.2 Vérifier PostgreSQL

psql --version    # Attendu : psql (PostgreSQL) 15.x
⚠️
Si psql est introuvable Ajoutez le dossier bin de PostgreSQL au PATH : C:\Program Files\PostgreSQL\15\bin (Variables d'environnement système → Path → Nouveau)

1.3 S'assurer que le service PostgreSQL est démarré

# PowerShell (admin) :
Get-Service postgresql*    # Vérifier le statut
Start-Service postgresql-x64-15    # Démarrer si nécessaire

# Ou depuis les Services Windows (services.msc) :
# Chercher "postgresql-x64-15" → clic droit → Démarrer

Base de données

Création de la base et du fichier .env

Si c'est la première fois que vous lancez le projet, vous devez créer la base de données et configurer le fichier server/.env.

Si vous avez déjà fait ce setup, passez directement à la section 3.

2.1 Créer la base de données

# Ouvrir psql :
psql -U postgres

# Dans le prompt psql :
postgres=# CREATE DATABASE helpdesk_dev;
postgres=# \q

2.2 Créer le fichier server/.env

Créez un fichier nommé .env dans le dossier server/ :

# ── Remplacez VOTRE_MOT_DE_PASSE par celui de votre compte postgres ──
DATABASE_URL=postgresql://postgres:VOTRE_MOT_DE_PASSE@localhost:5432/helpdesk_dev
JWT_SECRET=un-secret-fort-et-aleatoire-min-32-chars
PORT=3001
NODE_ENV=development
APP_URL=http://localhost:5173
UPLOADS_PATH=./uploads
COMPANY_NAME=Mon Helpdesk
SMTP_HOST=
SMTP_PORT=587
SMTP_USER=
SMTP_PASS=
SMTP_FROM=noreply@helpdesk.local
⚡
Astuce — Script automatique Vous pouvez aussi double-cliquer sur local-setup.bat à la racine du projet. Il crée la base, génère le .env, installe les dépendances et lance les migrations en une seule commande.

Setup du projet

Dépendances, migrations, seed

Ces étapes s'exécutent une seule fois (ou après un git pull majeur).

Option A — Script automatique (recommandé)

1
Double-cliquer sur local-setup.bat
À la racine du projet. Pas besoin de droits admin. Répond aux questions avec les valeurs par défaut (touche Entrée).
2
Attendre la fin du script
Le script installe les dépendances npm, applique les migrations Prisma et charge les données de test. Durée : 2 à 5 minutes.
3
Vérifier le message de fin
Le script doit afficher : "Setup terminé ! Lancez local-start.bat pour démarrer" avec les identifiants de test.

Option B — Manuellement (terminal)

Si le script automatique échoue, exécutez ces commandes depuis la racine du projet :

# 1 — Installer les dépendances
npm install                        # Racine
npm install --prefix server       # Backend
npm install --prefix client       # Frontend

# 2 — Générer le client Prisma
cd server
npx prisma generate

# 3 — Migrations + seed
npx prisma migrate deploy          # Créer les tables
npx prisma db seed                 # Charger les données de test
cd ..
✅
Seed réussi — données chargées Le seed crée automatiquement : 2 rôles (Admin, Agent), 4 utilisateurs, des catégories de tickets, des clients de démonstration et un modèle d'enquête de satisfaction.

Démarrer l'application

Backend + Frontend en parallèle

Méthode rapide

Double-cliquez sur local-start.bat à la racine du projet. Il démarre le backend et le frontend dans la même fenêtre.

# Ou depuis un terminal :
.\local-start.bat

# Ce que vous devez voir en quelques secondes :
  [server] Server running on port 3001
  [vite]   Local: http://localhost:5173/
🌐
Application disponible à
http://localhost:5173
Backend API : http://localhost:3001  ·  Ctrl+C pour arrêter

Méthode deux terminaux (alternative)

# Terminal 1 — Backend :
cd server  &&  npm run dev

# Terminal 2 — Frontend :
cd client  &&  npm run dev
⚠️
Si les ports 3001 ou 5173 sont déjà utilisés netstat -ano | findstr :3001 pour identifier le processus. Modifiez PORT=3002 dans server/.env si nécessaire.
🌐
Accès réseau local Le fichier vite.config.ts est configuré avec host: true. L'application frontend est donc accessible depuis d'autres PC du même réseau local sur le port 5173 (ex : http://192.168.x.x:5173). Cette option est active uniquement en mode développement.

Résolution des problèmes

Erreurs courantes et solutions rapides

❌ Page blanche ou "Cannot GET /"

Le frontend n'est pas démarré. Vérifiez que Vite tourne dans le terminal. Si le terminal est fermé, relancez local-start.bat.

❌ Erreur "connect ECONNREFUSED 127.0.0.1:3001"

Le backend est arrêté. Relancez le terminal et vérifiez que le message "Server running on port 3001" apparaît.

❌ Erreur de connexion — "Identifiants incorrects"

Le seed n'a pas été lancé ou a échoué. Relancez manuellement :

cd server
npx prisma db seed

❌ "DATABASE_URL must start with postgresql://"

Le fichier server/.env est absent ou mal configuré. Vérifiez que DATABASE_URL est bien renseigné et que le service PostgreSQL est démarré.

❌ "P1001 Can't reach database server"

# Démarrer PostgreSQL :
Start-Service postgresql-x64-15    # PowerShell admin

❌ Erreur TypeScript au démarrage du backend

Le client Prisma n'est pas généré. Lancez :

cd server
npx prisma generate
cd ..

❌ Upload de fichier échoue (erreur 500)

Le dossier server/uploads/ n'existe pas. Créez-le manuellement, ou vérifiez que UPLOADS_PATH=./uploads est bien dans server/.env.
