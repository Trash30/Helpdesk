Introduction

Objectif et perimetre de ce guide

Ce guide couvre le deploiement complet de l'application HelpDesk sur un mini PC ou poste sous Windows 10 ou Windows 11.

⚙️
Stack technique Node.js 20 + PostgreSQL 16 + PM2 (process manager) + Caddy (reverse proxy)
⏰
Duree estimee 45 a 60 minutes pour un deploiement complet.
1

Telechargements requis

Logiciels a installer sur Windows

Telechargez et installez les logiciels suivants :

  • ⚡
    Node.js 20 LTS Installer le .msi, cocher "Add to PATH"
    https://nodejs.org
  • 🐘
    PostgreSQL 16 Noter le mot de passe postgres lors de l'installation
    https://www.postgresql.org/download/windows/
  • 📁
    Git Accepter les options par defaut
    https://git-scm.com/download/win
  • 🌐
    Caddy (reverse proxy) Telecharger la version windows amd64
    https://caddyserver.com/download
  • 📝
    VS Code (optionnel) Pour editer les fichiers de configuration
    https://code.visualstudio.com
2

PM2 — Process Manager

PowerShell en Administrateur

Ouvrez PowerShell en tant qu'administrateur et executez :

npm install -g pm2
npm install -g pm2-windows-startup
🛈
pm2-windows-startup Ce package permet a PM2 de demarrer automatiquement avec Windows.
3

Base de donnees

Creation de l'utilisateur et de la base via pgAdmin ou psql

Ouvrez pgAdmin (installe avec PostgreSQL) → Query Tool, ou utilisez psql en ligne de commande :

CREATE USER helpdesk_user WITH PASSWORD 'ChangeMe123!';
CREATE DATABASE helpdesk_db OWNER helpdesk_user;
GRANT ALL PRIVILEGES ON DATABASE helpdesk_db TO helpdesk_user;
⚠️
Securite Remplacez ChangeMe123! par un mot de passe fort et unique.
4

Cloner le projet

Recuperation du code source
cd C:\
git clone https://github.com/Trash30/Helpdesk.git helpdesk
cd C:\helpdesk
5

Configuration — fichier .env

Variables d'environnement du serveur

Creez le fichier C:\helpdesk\server\.env avec le contenu suivant :

DATABASE_URL=postgresql://helpdesk_user:ChangeMe123!@localhost:5432/helpdesk_db
JWT_SECRET=SECRET-ALEATOIRE-32-CARACTERES-MINIMUM
PORT=3001
NODE_ENV=production
UPLOADS_PATH=C:/helpdesk/uploads
APP_URL=http://VOTRE-IP-OU-DOMAINE
💡
JWT_SECRET Vous pouvez generer un secret en PowerShell : [guid]::NewGuid().ToString() + [guid]::NewGuid().ToString()
6

Installation et build

Dependances, migration, seed, compilation
cd C:\helpdesk

# Installation des dependances
npm install --prefix server
npm install --prefix client

# Migration base de donnees + seed
cd server
npx prisma migrate deploy
npx prisma db seed
cd ..

# Build frontend + backend
npm run build --prefix client
npm run build --prefix server
✅
Verification Le build est reussi si aucun message d'erreur n'apparait. Les fichiers compiles du frontend se trouvent dans client\dist\.
7

Adapter ecosystem.config.js

Configuration PM2 pour Windows

Ouvrez ecosystem.config.js et remplacez toutes les references /opt/helpdesk par C:/helpdesk.

# Demarrage
pm2 start ecosystem.config.js
pm2-startup install
pm2 save
🛈
Verification Executez pm2 status pour verifier que le processus tourne. Le serveur ecoute sur le port 3001.
8

Caddy — reverse proxy

Configuration et installation en service

Placez caddy.exe dans C:\helpdesk\. Creez un fichier C:\helpdesk\Caddyfile :

:80 {
    handle /api/* {
        reverse_proxy localhost:3001
    }
    handle /uploads/* {
        root * C:/helpdesk
        file_server
    }
    handle {
        root * C:/helpdesk/client/dist
        file_server
        try_files {path} /index.html
    }
}

Testez puis installez en service :

cd C:\helpdesk
.\caddy.exe run --config Caddyfile    # tester d'abord
.\caddy.exe service install           # installer en service Windows
net start caddy
✅
Test Ouvrez http://localhost dans un navigateur. Vous devez voir la page de login HelpDesk.
9

Pare-feu Windows

Autoriser les connexions entrantes sur le port 80

En PowerShell administrateur :

New-NetFirewallRule -DisplayName "Helpdesk HTTP" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow
10

Exposition sur internet

Rendre l'application accessible depuis l'exterieur

Option A — Redirection de port (box internet)

Identique a la procedure Ubuntu : interface admin de la box → NAT/PAT → TCP port 80 externe vers IP locale du PC port 80.

Option B — Cloudflare Tunnel (recommande)

💡
Avantages SSL gratuit, aucun port a ouvrir, protection DDoS incluse.

Telechargez cloudflared-windows-amd64.msi depuis le site Cloudflare, puis suivez la meme procedure :

cloudflared tunnel login
cloudflared tunnel create helpdesk
cloudflared tunnel route dns helpdesk votre-sous-domaine.votre-domaine.com
cloudflared service install
11

Mises a jour

Procedure de mise a jour de l'application
cd C:\helpdesk
git pull origin main
npm install --prefix server && npm install --prefix client
cd server && npx prisma migrate deploy && cd ..
npm run build --prefix client && npm run build --prefix server
pm2 restart helpdesk-server
⚠️
Compte administrateur par defaut Apres le seed, les identifiants par defaut sont dans server/prisma/seed.ts. Changez le mot de passe immediatement apres la premiere connexion.
🎉
Deploiement termine ! Si toutes les etapes sont completes, votre instance HelpDesk est en production et accessible. Bonne utilisation !
